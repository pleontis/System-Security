============================================ process_packet ============================================
Καλείται μέσα στη main και είναι η κύρια συνάρτηση που επεξεργάζεται το πακέτο. Παίρνει ώς όρισμα το 
όνομα του αρχείου που θα ανοιχτεί. Αρχικά δημιουργείται ο handler και στη συνέχεια αφού μπορούν να 
διαβαστούν τα πακέτα εκτελείται μια while και γίνεται χρήση της pcap_next_ex ώστε να διαβαστούν ένα ένα 
τα πακέτα. Μέσα σε μια switch εάν το προτκόκολο του πακέτου πρόκειται για TCP ή UDP γίνεται περεταίρω 
επεξεργασία αλλιώς προχωράμε στο επόμενο πακέτο. Έπειτα ανακτώντας τα σωστά πεδία εκτυπώνουμε στο χρήστη 
τις πληροφορίες του πακέτου. Το προτόκολο βρίσκεται μέσω της συνάρτησης find_protocol. Στην συνέχεια έχοντας 
δημιουργήσει ένα array από structs, όπου το struct περιέχει τα 5 tuples τα οποία αποτελούν ένα network flow 
γίνεται σύγκριση για κάθε struct της λίστας με τα στοιχεία του τωρινού πακέτου. Εάν υπάρξει αντιστοιχία και 
είναι ΚΑΙ τα 5 ίδια τότε το network flow υπάρχει στη λίστα και προχωράμε στο επόμενο πακέτο. Επιπλέον πρέπει 
να ληφθεί υπόψιν η περίπτωση όπου ο αποστολέας και ο παραλλήπτης αλλάζουν ρόλους. Οι διευθύνσεις  ip και τα 
ports αντιστρέφονται και θεωρείται ότι δεν αλλάζει το network flow. Αλλιώς αυξάνεται ένας μετρητής ο οποίος 
δηλώνει αν πρόκειται για TCP flow ή UDP flow και γίνεται εισαγωγή του struct στο array.Τέλος εκτυπώνονται όσα 
ζητούνται στην εκφώνηση.
